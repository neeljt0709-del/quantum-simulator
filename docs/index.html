under the hood

Tensors, not 2ⁿ × 2ⁿ matrices

An n-qubit register is stored as a single flat statevector of length 2ⁿ. To apply a gate to one qubit without touching the rest, the simulator reshapes that flat vector into an n-axis tensor — one axis per qubit — and contracts the gate matrix against only the target axis.

This is mathematically identical to applying I ⊗ ⋯ ⊗ U ⊗ ⋯ ⊗ I to the full state, but it never builds that exponentially large matrix. The Bloch sphere above is the one-qubit special case of exactly this operation — once two qubits become entangled (via CNOT), there's no longer a single point on a sphere that represents either qubit alone.

API reference

Statevector(n)

MethodDescription
h(index)Apply Hadamard to qubit index
x(index) / y(index) / z(index)Apply Pauli gates
rotate_x(theta, index) / rotate_y(theta, index) / rotate_z(theta, index)Apply parameterized rotations
cnot(control, target)Apply controlled-NOT
reset(index)Reset a qubit to |0⟩
measure(index)Collapse and measure a qubit, returns the outcome (0 or 1)
probabilities()Returns |amplitude|² for every basis state
copy()Returns an independent deep copy of the statevector

Circuit(n)

MethodDescription
add_gate(gate, qubits, theta=0)Append a gate to the circuit. gate is one of h, x, y, z, cnot, reset, measure, rx, ry, rz. qubits is a list of target qubit indices; theta is required for rotation gates.
implement(sv)Runs the full gate sequence on a copy of sv, returns (results, final_statevector)
